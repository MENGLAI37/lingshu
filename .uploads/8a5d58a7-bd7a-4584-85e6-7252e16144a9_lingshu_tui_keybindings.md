# LingShu TUI 快捷键方案重构设计

> 版本: v1.0 | 设计日期: 2026-07-01
> 设计目标: 完全忽略现有快捷键，基于主流 TUI 工具（Vim / Tmux / Ranger / Less）惯例重新设计

---

## 一、交互层级架构

本方案将界面状态划分为 **6 个互斥层级**，高优先层级会拦截快捷键，不向下层穿透，避免冲突。

| 层级 | 名称 | 说明 |
|------|------|------|
| L0 | 系统级 | 任何状态下均可触发（使用 `Ctrl+` / `Alt+` 组合，避免与输入冲突） |
| L1 | Normal 模式 | 主界面默认状态，输入框未聚焦 |
| L2 | Insert 模式 | 输入框聚焦，用于文本编辑 |
| L3 | 命令预览弹窗 | L2 工具执行前的确认对话框（模态） |
| L4 | 代码查看器弹窗 | JSON/YAML/Table 高亮内容查看器（模态） |
| L5 | 配置面板弹窗 | LLM Provider 配置管理（模态，含列表/编辑子模式） |
| L6 | 帮助覆盖层 | 快捷键帮助浮层（非模态，可与其他层级叠加） |

---

## 二、L0 — 系统级快捷键（最高优先级）

> 仅使用 `Ctrl+` / `Alt+` / `F1~F12` 组合，确保在输入框内也不会误触。

| 快捷键组合 | 触发场景 | 对应功能 | 设计依据 |
|-----------|---------|---------|---------|
| `Ctrl+C` | **全局** | 强制退出程序（发送 SIGINT） | Unix 终端标准信号 |
| `Ctrl+Q` | 全局（非 Insert 模式） | 优雅退出程序 | Emacs / Tmux 退出惯例；避免单键 `q` 的误触问题 |
| `F1` / `?` | 全局（非 Insert 模式） | 显示 / 隐藏帮助覆盖层 | `F1` 是跨平台通用帮助键，`?` 是 Vim / Ranger / Tmux 惯例 |
| `Esc` | 全局（存在弹窗时） | 关闭最上层弹窗，逐层返回 | 模态对话框的通用取消键 |

---

## 三、L1 — Normal 模式（主界面 / 聊天视图）

> 进入方式：程序启动默认进入，或从 Insert 模式按 `Esc` 返回。

| 快捷键组合 | 触发场景 | 对应功能 | 设计依据 |
|-----------|---------|---------|---------|
| `i` / `Enter` | Normal 模式 | 聚焦输入框，进入 **Insert 模式** | Vim 中 `i` = insert；`Enter` 对新手更直觉 |
| `j` / `↓` | Normal 模式 | 聊天区域向下滚动一行 | Vim 导航核心键位 |
| `k` / `↑` | Normal 模式 | 聊天区域向上滚动一行 | Vim 导航核心键位 |
| `Ctrl+D` / `PgDn` | Normal 模式 | 聊天区域向下翻半页 | Vim `Ctrl+D`（Down half-page） |
| `Ctrl+U` / `PgUp` | Normal 模式 | 聊天区域向上翻半页 | Vim `Ctrl+U`（Up half-page） |
| `g` / `Home` | Normal 模式 | 跳转到聊天记录顶部 | Vim `gg` 的简化版 |
| `G` / `End` | Normal 模式 | 跳转到聊天记录底部 | Vim `G`（Go to bottom） |
| `/` | Normal 模式 | 打开聊天内容搜索（功能需求说明） | Vim `/` = 搜索 |
| `n` | Normal 模式 | 跳到下一个搜索结果 | Vim 搜索导航 |
| `N` | Normal 模式 | 跳到上一个搜索结果 | Vim 搜索导航 |
| `c` | Normal 模式 | 打开 **L5 配置面板** | **C**onfig 首字母 |
| `t` | Normal 模式 | 循环切换主题（暗色 → 亮色 → 高对比度） | **T**heme 首字母 |
| `r` | Normal 模式 | 重新初始化 Agent Loop（刷新连接） | **R**eload / **R**efresh 首字母 |
| `m` | Normal 模式 | 切换输入模式（单行 ↔ 多行）（功能需求说明） | **M**ode 首字母 |

## 四、L2 — Insert 模式（输入框聚焦）

> 进入方式：在 Normal 模式下按 `i` 或 `Enter`。此模式下绝大部分按键用于文本编辑，仅保留少量受控快捷键。

| 快捷键组合 | 触发场景 | 对应功能 | 设计依据 |
|-----------|---------|---------|---------|
| `Enter` | Insert 模式 | 发送消息 | 通用即时通讯 / 聊天工具惯例 |
| `Shift+Enter` | Insert 模式 | 换行（不发送） | 通用多行输入惯例 |
| `Ctrl+Enter` | Insert 模式 | 发送消息（备选方案） | 部分用户更习惯的组合 |
| `Esc` | Insert 模式 | 退出聚焦，返回 **L1 Normal 模式** | Vim Esc = 退出插入模式 |
| `Ctrl+P` / `↑` | Insert 模式 | 上一条输入历史 | Shell / Readline 历史导航 |
| `Ctrl+N` / `↓` | Insert 模式 | 下一条输入历史 | Shell / Readline 历史导航 |
| `Ctrl+L` | Insert 模式 | 清空当前输入框 | Shell `clear` 命令映射 |
| `Ctrl+A` | Insert 模式 | 光标移至行首 | GNU Readline 标准 |
| `Ctrl+E` | Insert 模式 | 光标移至行尾 | GNU Readline 标准 |
| `Ctrl+K` | Insert 模式 | 删除光标至行尾的内容 | GNU Readline 标准 |
| `Ctrl+U` | Insert 模式 | 删除整行 | GNU Readline 标准 |
| `Ctrl+W` | Insert 模式 | 删除光标前的一个单词 | GNU Readline 标准 |

| `N` / `Esc` | 命令预览弹窗 | **取消执行**命令 | Y/N 对话框通用惯例 |
| `Tab` / `→` | 命令预览弹窗 | 焦点切换到下一个按钮 | 焦点导航惯例 |
| `Shift+Tab` / `←` | 命令预览弹窗 | 焦点切换到上一个按钮 | 焦点导航惯例 |
| `j` / `↓` | 命令预览弹窗 | 向下滚动命令详情（内容溢出时） | Vim 导航 |
| `k` / `↑` | 命令预览弹窗 | 向上滚动命令详情 | Vim 导航 |
| `v` | 命令预览弹窗 | 查看完整影响分析报告（功能需求说明） | **V**iew 首字母 |

## 十一、防冲突与防误触设计说明

### 11.1 为什么弃用单键 `q` 退出？
现有实现中，在 Normal 模式下按 `q` 会直接退出。这在以下场景极易误触：
- 用户从其他应用切换回来，手放在 `q` 键附近；
- 用户想输入以 `q` 开头的命令，误以为自己在 Insert 模式。

**改进**：使用 `Ctrl+Q` 替代，既保留了快速退出的能力，又彻底消除了误触风险。

### 11.2 为什么输入框内不响应 `?` / `c` / `t` 等全局键？
Insert 模式下，所有单字符键（包括 `?`、`c`、`t`）都优先作为文本输入，只有 `Ctrl+` / `Alt+` / `Esc` / `Enter` 组合被拦截为快捷键。这遵循了所有主流编辑器（Vim、Emacs、VS Code 终端）的惯例。

### 11.3 模态弹窗的 Esc 行为
当多个弹窗同时存在时（理论上不应发生），`Esc` 仅关闭最上层弹窗，逐层回退，而非一次性回到主界面。这符合用户"撤销一步操作"的心理模型。

### 11.4 方向键与 Vim 键并存
所有导航操作同时支持 `j/k` 和 `↑/↓`（以及对应的其他方向键），既满足 Vim 资深用户的肌肉记忆，也不为难只习惯方向键的新手。
